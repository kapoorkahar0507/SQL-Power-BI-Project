# Retail Sales Analysis Project (SQL + Power BI)

## Project Overview

This project demonstrates a complete **Data Analytics workflow**, starting from **database creation in SQL Server** to **data analysis and visualization in Power BI**.

The objective of the project is to analyze **retail sales performance**, evaluate **product profitability**, and understand the **impact of promotions on sales** using structured data.

The project includes:

* Database design and table creation using SQL
* Data generation and data analysis using SQL queries
* Interactive dashboard development in Power BI
* Business insights derived from the data

---

# 1. Database Design (SQL Server)

A relational database was created using **Microsoft SQL Server** consisting of four main tables:

### Customers_Table

Stores customer information.

Fields:

* Customer_ID (Primary Key)
* Customer_Name
* City
* State
* Pincode
* Email
* Phone_Number

---

### Product_Table

Contains product-related information.

Fields:

* Product_ID (Primary Key)
* Product_Name
* Product_Line
* Price

---

### Promotion_Table

Contains details of promotional campaigns.

Fields:

* Promotion_ID (Primary Key)
* Promotion_Name
* Ad_Type
* Coupon_Type
* Offer

---

### Sales_Table

Stores transaction records and connects all tables using foreign keys.

Fields:

* Sale_Date
* Customer_ID (Foreign Key)
* Product_ID (Foreign Key)
* Promotion_ID (Foreign Key)
* Unit_Sold

This table represents the **fact table** of the dataset.

---

# 2. Data Generation

To simulate real business transactions, **2000 sales records** were generated using SQL.

Key characteristics of the generated data:

* Customer_ID values between **1–50**
* Product_ID values between **P001–P050**
* Promotion_ID values between **PR001–PR005**
* Approximately **1700 rows contain NULL promotions** to simulate sales without marketing campaigns
* Randomized sale dates and unit quantities

SQL functions used for random data generation:

* NEWID()
* CHECKSUM()
* ROW_NUMBER()
* DATEADD()

---

# 3. Data Analysis Using SQL

Multiple SQL queries were performed using **JOIN operations** to analyze the dataset.

Key analyses performed:

### Sales and Revenue Analysis

* Total revenue generated by each product
* Top and bottom performing products
* Revenue contribution by product line

### Customer Analysis

* Total purchase value per customer
* Customer purchase frequency
* High-value customers

### Regional Sales Analysis

* Revenue distribution by state
* Regional demand patterns

### Promotion Performance

* Sales with promotion vs without promotion
* Most effective promotion campaigns

### Trend Analysis

* Monthly sales trend
* Units sold over time

These analyses helped generate insights used later in the dashboard.

---

# 4. Power BI Dashboard

The dataset was connected to **Power BI** to create an interactive dashboard for business analysis.

The dashboard includes the following visualizations:

### Top and Bottom Products

Identifies the best and worst performing products based on revenue and units sold.

### Profit vs Sales

Compares sales revenue and profit to evaluate product profitability.

### Discount by Promotion Category

Analyzes how different promotional strategies impact sales.

### Sales Trend by Period

Visualizes sales performance over time to identify trends and seasonality.

### Period Comparison Analysis

Compares two selected time periods to analyze changes in:

* Sales
* Profit
* Units Sold

This helps measure business performance between different time ranges.

# 6. Project Insights

From the analysis, the following insights can be derived:

* Certain products generate the majority of revenue.
* Promotional campaigns influence sales but many purchases occur without promotions.
* Sales performance varies across regions.
* Seasonal patterns can be observed in sales trends.

These insights can help businesses make better decisions in **marketing strategy, inventory management, and customer targeting**.
